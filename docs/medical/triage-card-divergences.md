## 一图看懂:两个系统性偏差

1. **红旗通胀(triage 偏严)**:triage 把卡里定为 `urgent_vet`(黄向、不急停)的信号,常常升格成「红 + 急停」。方向安全、不漏判,但会过度急停、稀释红档可信度,并可能在中途急停时**挤掉对更危险信号的追问**。
2. **宿主风险漏判(triage 偏松)**:多条 flow **不问年龄 / 性别 / 疫苗 / 体型 / 慢病**,导致卡里依赖宿主分层的硬急症无法触发。**这是真实漏诊,后果最重。**

## 四、NO_GREEN vs 卡绿档不一致(Category C —— triage 更保守,安全但与真值源字面冲突)

Category C 是「方向偏安全但与卡字面冲突」,不是漏判。需决定:以卡为真值源采纳窄绿档,还是保留 triage 的更保守封绿(并在卡的 `§10 来源分歧` 里记录这是产品有意更严)。

## 五、结构性建议(收敛 Category A 的根本)

多条 flow 缺**宿主风险采集**,导致卡的 host_risk 分层无处落地。建议补的题(均有卡 claim 支撑):

- **pee**:加「是不是公猫」→ 命中 `uo_005`,公猫 + 尿少即升红。
- **noeat**:加「年龄(是否 ≤6 周幼猫)/ 是否肥胖」→ `ano_002` / `ano_004`。
- **mouth**:加「突发大量流口水 + 未免疫或被流浪动物咬过」红旗 → `oral_014`。
- **breath**:把「有点费劲」升 redFlag(`bre_rf_labored`);给 breathFlow 补挂通用红旗组(覆盖 `bre_rf_collapse`)。
- **skin**:跳蚤选项与「幼猫虚弱/牙龈苍白」联动红旗 → `skin_014 + emg_009`;并补挂通用红旗组。
- **eye**:外伤红旗扩入「化学品/异物入眼」→ `eye_rf_trauma_chemical`。
- **blood**:补「腹部膨大」「异物插入」红旗 → `bld_rf_internal` / `bld_rf_object`。
- **general/lethargy**:补牙龈颜色(含黄疸)、脱水、不吃时长、幼猫疫苗采集,收紧绿档前置。
- **多条**:`RED_FLAG_OPTIONS` 的「牙龈苍白」补「发黄(黄疸)」;考虑补一条脱水信号(皮肤回弹慢/眼窝凹)。
